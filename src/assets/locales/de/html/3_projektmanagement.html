Standardprotokoll zur De-Identifikation nach DICOM

In PS 3.15 Anhang E, spezifiziert der DICOM-Standard eine Reihe von De-Identifikationsregeln für den Einsatz in verschiedenen Situationen. Die Regeln sind in ein Basisanwendungsvertraulichkeitsprofil gruppiert, das alle geschützten Gesundheitsinformationen (PHI) entfernt, und in eine Reihe von Optionen, die bestimmte Klassen von PHI in bestimmten Fällen lockern können. Die folgenden Abschnitte beschreiben das von dem Anonymisierungsprogramm implementierte De-Identifikationsprotokoll unter Verweis auf das DICOM-Profil und seine Optionen.

113100: Basisanwendungsvertraulichkeitsprofil

Das RSNA DICOM-Anonymisierungsprogramm implementiert das Basisprofil. Das Programm importiert die Standortkennung (SiteID und UIDRoot) entweder aus einer Excel-Indexdatei, die vom Java-Anonymisierer exportiert wurde, oder generiert automatisch eine neue eindeutige SiteID und verwendet eine Standard-UIDRoot. Die Elemente PatientName (0010,0010) und PatientID (0010,0020) werden durch Zuweisen sequenzieller Integer zu eindeutigen PHI-Patienten-IDs modifiziert, wenn sie erkannt werden. Der sequenzielle Integer wird dem SiteID-Parameter hinzugefügt und ergibt eine Zeichenfolge im Format [SiteID]-[sequenzieller Integer], die zur Befüllung beider Elemente verwendet wird. Wenn die PHI-Patienten-ID fehlt oder leer ist, weist das Programm den Elementen PatientID und PatientName [SiteID]-000000 zu. Das Programm verwaltet eine Datenbank, um sicherzustellen, dass alle Studien desselben Patienten auf dieselbe Weise de-identifiziert werden.
  1. UIDs werden durch solche ersetzt, die im Format generiert sind: [UIDRoot].[SiteID].[sequenzieller Integer]
  2. Alle UIDs werden in Gruppen entfernt.
  3. Alle Elemente in den Gruppen 0032 bis 4008 werden entfernt.
  4. Alle privaten Gruppen werden entfernt.
  5. Das Element PatientIdentityRemoved (0012,0062) wird mit dem Wert "YES" versehen.
  6. Das Element DeIdentificationMethod (0012,0063) wird mit dem Wert "RSNA DICOM ANONYMIZER" versehen.
  7. 113100 wird dem Element DeIdentificationMethodCodeSequence (0012,0064) hinzugefügt.
  8. Ein privater Block: 0x0013 wird erstellt und mit dem Wert "RSNA" versehen, der Projektname und die SiteID werden diesem Block hinzugefügt.

113101: Option zur Bereinigung von Pixel-Daten

Nicht implementiert.

113102: Option zur Bereinigung erkennbarer visueller Merkmale

Nicht implementiert.

113103: Option zur Bereinigung von Grafiken

Nicht implementiert, jedoch werden Gruppen von Kurven und Überlagerungen (5XXX und 6XXX) entfernt.

113104: Option zur Bereinigung strukturierter Inhalte

Nicht implementiert, jedoch können Projekteinstellungen die Speicherung von SR (Structured Report)-bezogenen Klassen erlauben/verbieten.

113105: Option zur Bereinigung von Deskriptoren

Diese Option ist teilweise implementiert. Es werden nur die folgenden durch die Option definierten Elemente behalten:

  • (0008,1030) StudyDescription
  • (0008,103e) SeriesDescription

113106: Option zur Beibehaltung vollständiger longitudinale zeitlicher Informationen

Nicht implementiert.

113107: Option zur Beibehaltung geänderter longitudinaler zeitlicher Informationen

Diese Option ist teilweise implementiert.

  • Alle Daten werden durch Berechnung des MD5-Hashs von PHI PatientID geändert.
  • Der ganze Wert des Hashs wird durch die Anzahl der Tage in 10 Jahren (3652) geteilt und der Rest wird verwendet, um das Datum zu verschieben, das gerade geändert wird.
  • Dies ermöglicht es, die zeitlichen Beziehungen zwischen Studien eines einzelnen Patienten zu bewahren, während für verschiedene Patienten unterschiedliche Verschiebungen verwendet werden.
  • Die Elemente der Uhrzeit (Tageszeit) werden nicht geändert.
  • 113107 wird dem Element DeIdentificationMethodCodeSequence (0012,0064) hinzugefügt.

113108: Option zur Beibehaltung der Patientenmerkmale

Diese Option ist teilweise implementiert. Es werden nur die folgenden durch die Option definierten Elemente behalten:

  • (0010,0040) PatientSex
  • (0010,1010) PatientAge
  • (0010,1020) PatientSize
  • (0010,1030) PatientWeight
  • (0010,2160) EthnicGroup
  • (90010,21a0) SmokingStatus
  • 113108 wird dem Element DeIdentificationMethodCodeSequence (0012,0064) hinzugefügt.

113109: Option zur Beibehaltung der Geräteidentität

Diese Option ist teilweise implementiert. Es werden nur die folgenden durch die Option definierten Elemente behalten:

  • (0008,0070) Manufacturer
  • (0008,1090) ManufacturerModelName
  • 113109 wird dem Element DeIdentificationMethodCodeSequence (0012,0064) hinzugefügt.
